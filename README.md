# Лабораторная_А_XOR-­связный список 
## Вариант 12

XOR-­связный список — структура данных, похожая на обычный двусвязный список, однако в каждом
элементе хранящая только один адрес — результат выполнения операции XOR над адресами
предыдущего и следующего элементов списка. Для того, чтобы перемещаться по списку, необходимо взять два последовательных адреса и выполнить над ними операцию XOR, которая и даст реальный адрес
следующего элемента.

Реализуйте XOR­-связный список элементами которого являются ASCII­Z строки, со следующими
операциями:

1. Добавить элемент в список
2. Удалить элемент из списка по ключу
3. Удалить данный элемент списка (по адресу элемента)
4. Найти элемент по ключу
5. Итерирование по списку
   
В чем недостатки и преимущества XOR-связного списка?

# Ответы на вопросы

## Недостатки XOR-связного списка
1. Усложненная навигация: для передвижения по списку необходимо знать текущий узел и значение его XOR-указателя. Без доступа к предыдущему узлу трудно ориентироваться, что усложняет обход структуры.
2. Отладка: в случае повреждения указателей сложно восстановить состояние структуры, также сложно отслеживать структуру списка.
3. Редактирование: добавление и удаление узлов может увеличить риск ошибок по сравнению с односвязным списком.

## Преимущества XOR-связного списка
1. Экономия памяти: используется меньше указателей (1 вместо 2 на узел).
2. Компактность: эффективное использование памяти при больших объемах данных.
3. Обратимость: позволяет передвигаться по списку в обоих направлениях при наличии одного указателя и начальной точки.
